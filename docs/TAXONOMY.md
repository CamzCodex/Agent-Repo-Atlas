# Taxonomy

The atlas uses a small controlled vocabulary to keep search and scoring stable.

## Primary types

- `library`
- `framework`
- `template`
- `ui-system`
- `mcp-server`
- `agent-framework`
- `design-reference`
- `implementation-pattern`
- `tooling`

## Acquisition modes

- `reference`
- `dependency-candidate`
- `fork-candidate`
- `sandbox-research`
- `reject`

## Maintenance states

- `active`
- `slow-moving`
- `stale`
- `archived`

## Risk flags

- `license-unclear`
- `unmaintained`
- `security-sensitive`
- `low-documentation`
- `large-surface-area`
- `rapidly-changing`

## Tagging guidance

- Prefer concrete technology names over broad marketing labels.
- Use tags that help an agent search by task, stack, and artifact type.
- Keep tags short and normalized.
